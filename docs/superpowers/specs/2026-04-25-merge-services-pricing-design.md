# Объединение блоков «Услуги» и «Цены»

## Контекст

Сейчас на лендинге два соседних блока: `Services` (4 карточки с описанием услуг) и `Pricing` (таблица из 3 строк с ценами). Услуги и цены частично пересекаются, но не на 100% (в Pricing нет «Стяжки с тёплым полом»). Решено объединить их в один блок, в котором у каждой услуги есть цена.

## Что меняется

**Новый объединённый блок `Services`** (компонент остаётся, файл `src/components/Services/index.tsx`):

- Заголовок секции: «Услуги», иконка `Trowel` (как сейчас).
- Сетка 4 карточек: `sm:grid-cols-2 lg:grid-cols-4` (как сейчас).
- В каждой карточке:
  - title (`font-display uppercase text-xl`)
  - description (`text-text/70 leading-relaxed`)
  - разделитель `border-t-2 border-border` с верхним отступом
  - цена: `font-display text-3xl text-accent`, формат «от 450 ₽/м²»
- Под сеткой — сноска: «Цены ориентировочные. Точную смету пришлю после замера.» (переезжает из Pricing).

**Конфиг `src/config/site.ts`:**

- Тип `services` расширяется: `{ title: string; description: string; pricePerM2: number; note?: string }[]`.
- 4-я услуга «Стяжка с тёплым полом» получает `pricePerM2: 530, note: 'от'` (450 базовая полусухая + 80 надбавка за тёплый пол из `pricing.extras.overUnderfloor`).
- `pricing.items` удаляется (больше не отображается).
- `pricing.rates`, `pricing.thicknessMultiplier`, `pricing.extras` остаются — их использует калькулятор в `src/lib/pricing.ts` для квиза.

**`src/App.tsx`:**

- Удаляется import `Pricing` и его рендер.

**Файлы на удаление:**

- `src/components/Pricing/index.tsx`

## Что НЕ меняется

- Логика расчёта в `src/lib/pricing.ts` и квиз.
- Брутальный стиль (border-2, accent-color, font-display).
- Все остальные секции лендинга.

## Тестирование

Визуальная проверка после `bun run dev`:
- Один блок вместо двух, без разрыва в скролле.
- Все 4 услуги отображаются с ценой.
- На мобильном (1 колонка), планшете (2), десктопе (4) карточки выглядят корректно.
- Квиз по-прежнему считает цену.
